---------------------------------------------------------------------------------------------------------------------------------------------

ARRAY
-------
- Sequence
  Elements in sequence containers are ordered in a strict linear sequence. Individual elements are accessed by their position in this       
  sequence.
- Contiguous storage
  The elements are stored in contiguous memory locations, allowing constant time random access to elements. Pointers to an element can be 
  offset to access other elements.
- Fixed-size aggregate
  The container uses implicit constructors and destructors to allocate the required space statically. Its size is compile-time constant. No   
  memory or time overhead.

---------------------------------------------------------------------------------------------------------------------------------------------

VECTOR
--------
Sequence
  Elements in sequence containers are ordered in a strict linear sequence. Individual elements are accessed by their position in this       
  sequence.
Dynamic array
  Allows direct access to any element in the sequence, even through pointer arithmetics, and provides relatively fast addition/removal of 
  elements at the end of the sequence.
Allocator-aware
  The container uses an allocator object to dynamically handle its storage needs.
Notes
  Vector allocate extra storage for possible growth.
  One big memory block with all elements next to each other (BETTER FOR CPU AND SPEED ACCESS) :
    -If needed allocate a NEW vector and move all values to this one (more expensive)
+
  Efficient accessing his elements, and adding/removing end elements
-
  adding/removing elements other than end

---------------------------------------------------------------------------------------------------------------------------------------------

DEQUE
------
Sequence
  Elements in sequence containers are ordered in a strict linear sequence. Individual elements are accessed by their position in this 
  sequence.
Dynamic array
  Generally implemented as a dynamic array, it allows direct access to any element in the sequence and provides relatively fast addition/
  removal of elements at the beginning or the end of the sequence.
Allocator-aware
  The container uses an allocator object to dynamically handle its storage needs.--
Notes
  Like vector, but can be expanded or contracted from begin and end.
  Accessing with dereferencing iterator cause undefined behavior
  Stocked as chunks of memory dispersed in the memory
    - If needed just add one more chunks and keep its location in a map array (ptr array)
+
  More efficient to grow than VECTOR and less costly on reallocation
-
 adding/removing elements other than end/beginning
 less consistent iterators and references than lists and forward lists

VECTOR vs DEQUE
----------------
Cache CPU (Vector meilleur)
V : Memoire contigu permet de au CPU stocker en cache tous les elements
D : sauter de chunk en chunk -> doit retourner dans la RAM (plus long)

Cout des operations (Vector meilleur)
V : vector[i] -> prend l'adresse de base et fait une ADDITION (tres peu couteux, un cycle d'horloge (pacompri))
D : deque[i]  -> Division pour trouver le bon chunk, trouver l'index du pointer dans la map array, lire la donnee

Empreinte memoire (Vector meilleur)
V : tres leger, 3 pointeurs (debut, fin des donnees, fin de la capacite alloue)
D : Stock la map array (pointeurs), chunks pas forcement pleins (memoire pour rien)

Compatibilite avec le C (Vector meilleur)
(Souvent requis d'avoir des tableau de memoire contigu (tous a cote))
V : Lisible
D : impossible detre lu car donnees fragmentees/dispersees

Insertion ou suppression massive (Deque meilleur)
V : TOUT bouger et reallouer
D : Ajoute un chunk et rempli

Systeme FIFO (First in, First out) (Deque meilleur)

Proteger pointeurs (Deque meilleur)
V : int *ptr = &vector[5] devient mort si tu realloue le vector
D : ptr garder car on supprime pas les anciens chunks

Grosse quantite de donnee
V : Trouver 2go de memoire contigu tres dur
D : Etaler dans pleins d emplacement memoire

---------------------------------------------------------------------------------------------------------------------------------------------

FORWARD LIST
------
Sequence
  Elements in sequence containers are ordered in a strict linear sequence. Individual elements are accessed by their position in this 
  sequence.
Linked list
  Each element keeps information on how to locate the next element, allowing constant time insert and erase operations after a specific 
  element (even of entire ranges), but no direct random access.
Allocator-aware
  The container uses an allocator object to dynamically handle its storage needs.
Notes
  Better for inserting, extracting, and moving elements in any position
  more efficient (memory wise) than list
  Extra memory to keep address of next element


---------------------------------------------------------------------------------------------------------------------------------------------

LIST
------
Sequence
  Elements in sequence containers are ordered in a strict linear sequence. Individual elements are accessed by their position 
  in this sequence.
Doubly-linked list
  Each element keeps information on how to locate the next and the previous elements, allowing constant time insert and erase 
  operations before or after a specific element (even of entire ranges), but no direct random access.
Allocator-aware
  The container uses an allocator object to dynamically handle its storage needs.
Notes
  Better for inserting, extracting, and moving elements in any position
  Extra memory to keep address of next and previous element
